# Song Name to LED

Spotify에서 현재 재생 중인 노래 제목을 가게 LED 전광판에 실시간으로 표시하는 프로젝트.

## 현재 상태
**계획 단계** — 전광판 프로토콜 분석 필요

## 장비 정보
| 항목 | 내용 |
|------|------|
| LED 전광판 컨트롤러 | Huidu 추정 (뒷면 확인 불가) |
| 제어 프로그램 | HDPlayer |
| 연결 방식 | 전광판 자체 와이파이(AP 모드)에 직접 연결 |
| 가게 인터넷 | 공유기 와이파이 |

## 진행 단계

### 1단계: 전광판 프로토콜 분석 (가게에서 진행)
1. 가게 노트북에 **Wireshark** 설치 (무료, https://www.wireshark.org)
2. 전광판 와이파이에 연결
3. Wireshark에서 캡처 시작
4. HDPlayer로 전광판에 테스트 글씨 전송
5. Wireshark 캡처 중지 → `.pcap` 파일 저장
6. 캡처 파일을 분석하여 전광판의 IP 주소, 포트, 데이터 형식 파악

**이 단계가 핵심** — 프로토콜을 알아야 프로그램 없이 직접 명령을 보낼 수 있음.

### 2단계: Spotify 현재 재생곡 가져오기
- **방법 A (간단)**: Spotify 데스크톱 앱의 창 제목 읽기 (Python)
- **방법 B (정석)**: Spotify Web API 사용 (OAuth 인증 필요, 더 안정적)
- 방법 A 먼저 시도

### 3단계: 자동 전송 스크립트
- Python 스크립트: 5~10초마다 Spotify 재생곡 확인
- 노래가 바뀌면 → LED 전광판에 새 제목 전송
- 백그라운드 자동 실행

### 4단계: 네트워크 구성
- **USB 와이파이 어댑터** 또는 **유선랜**으로 인터넷 연결 유지
- 나머지 와이파이는 전광판 전용
- 두 네트워크 동시 연결 상태에서 스크립트 실행

## 필요 장비
- USB 와이파이 어댑터 (1~2만원) — 인터넷과 전광판 와이파이 동시 연결용
  - 또는 유선랜(이더넷)으로 인터넷 연결 가능하면 불필요

## 리스크
- Huidu 프로토콜이 암호화되어 있으면 직접 전송 불가 → 대안: HDPlayer 자동화 (pyautogui)
- 전광판 모델에 따라 프로토콜이 다를 수 있음

## 대안: HDPlayer 자동화
프로토콜 분석이 안 될 경우, HDPlayer 프로그램 자체를 자동 조작하는 방법:
- Python + pyautogui로 HDPlayer의 텍스트 입력 → 전송 버튼 클릭 자동화
- 덜 안정적이지만 프로토콜 몰라도 가능
